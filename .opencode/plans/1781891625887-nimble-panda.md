## What's Needed

### 1. Switch to `@sveltejs/adapter-vercel` explicitly
`adapter-auto` works but using `adapter-vercel` directly gives:
- Version stability (adapter-auto pins at build time)
- Access to Vercel-specific config (ISR, regions, image optimization)
- Slightly faster CI (no platform detection step)

**Files to change:**
- `vite.config.ts`: change `adapter-auto` import → `@sveltejs/adapter-vercel`
- `package.json`: can remove `adapter-auto` (optional cleanup)

### 2. Vercel project setup (done once in dashboard)
- Push repo to GitHub (`origin` already set to `github.com:fm39hz/fm39hz-blogs.git`)
- Import project in Vercel dashboard (or use `vercel` CLI)
- Vercel auto-detects SvelteKit, uses `bun run build` as build command
- Output directory: `.svelte-kit/output` (handled automatically by adapter)

### 3. Domain configuration
- `config.ts` already has `url: 'https://fm39hz.online'`
- Need to add `fm39hz.online` domain in Vercel dashboard → Project Settings → Domains
- Configure DNS at the domain provider (CNAME to `cname.vercel-dns.com`)

### 4. Deployment strategy options

| Option | Pros | Cons |
|---|---|---|
| **Fully static** (current) | Fastest, zero cold starts, free tier | Content change requires re-deploy |
| **ISR** (Incremental Static Regeneration) | Content updates without re-deploy | Needs server function (edge/Node) |

**Recommendation:** Keep fully static for now. This blog content is markdown committed to the repo — changes imply a git push → auto-deploy anyway. ISR adds complexity with no benefit here.

### 5. CI/CD (auto-deploy on push)
- Vercel GitHub integration: connect repo → every push to `main` deploys production
- Preview deployments for PR branches (automatic with Vercel + GitHub)
- No manual CI config needed — Vercel handles this

### 6. Optional enhancements (post-deploy)
- **Web Analytics**: install `@vercel/analytics`, add to `+layout.svelte`
- **Speed Insights**: enable in Vercel dashboard
- **Skew Protection**: enable in Vercel project settings (prevents version mismatch for active users)
